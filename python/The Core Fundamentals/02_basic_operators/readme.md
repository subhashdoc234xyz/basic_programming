2. Data Structures

3. Modular Code

4. Advanced Foundations (Once you are comfortable)

Pro-Tips for your GitHub Repository
Include a README.md: In the root of your repository, write a brief explanation of what the project is. In each sub-folder, add a small README.md explaining the specific concept covered there.

Consistent Naming: Use snake_case for your Python files (e.g., calculate_area.py, not CalculateArea.py).

Comment your code: Don't just write the code—write comments explaining why you did it that way. It helps your future self understand your logic.
